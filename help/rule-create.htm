Create Rule

The 'Create a rule...' dialog is displayed by pressing the '+' below the rule list box on the main program screen. The user is then presented with the dialog box below.



The screen is divided into 4 sections:

Rule Settings Area
Rule Subjects Area
Rule Exclusion Area
Rule Verbs Area
Rule Buttons

Rule Settings

The topmost part of the dialog box is where the user sets the overall rule settings. Here the user is presented with the folder to which the rule is assigned, and allowed to create a description, or rule title. Here the user can also apply the following rule options:


  • Enabled - If checked, the specific rule will be applied, and potentially act upon the files within the folder to which this rule is assigned. This option is also displayed on the main screen in the left most column of the right side rule box. A rule may be enabled/disabled from either location within the application.
  • Confirm Action - If checked, a dialog box with be displayed, confirming the action(s) of the rule prior to executing these actions. If a confirmation is actively being displayed, the program will essentially be 'paused' until user input is provided. A user may either confirm or deny the actions of the particular rule in question.
  • Recursive - If checked, the program will step through all the subdirectories of the folder to which the rule is assigned. If left unchecked, the rule in question will only operate on files within the immediate folder, and will not step into folders deeper within the tree structure.

Rule Subjects

The Rule Subjects area is where the user lists the conditions or files on which the rule will act.


The following rule subjects, and their corresponding options are available to be queried against all files within the defined folder:

  • Name - This subject acts upon the filename of the file in question, excluding the file extension.
    • is - acts upon a file if the filename matches exactly.
    • is not - acts upon all files that do not match the filename exactly .
    • matches one of - acts upon a file if the filename matches any one of the elements in a comma separated list. NOTE: The list must not contain spaces between the elements and the comma, otherwise you may miss a match.
    • does not match one of - acts upon a file if the filename does not match any one of the elements in a comma separated list. NOTE: The list must not contain spaces between the elements and the comma, otherwise you may miss a match.
    • contains - acts upon a file if the filename contains the specified text exactly.
    • does not contain - acts upon a file if it does not contain the specified text exactly.
    • contains one of - acts upon a file if the filename contains any one of the elements in a comma separated list. NOTE: The list must not contain spaces between the elements and the comma, otherwise you may miss a match.
    • RegEx - acts upon a file if the filename matches the Regular Expression inserted. For basic Regular Expression syntax, please used the following reference page: http://www.autohotkey.com/docs/misc/RegEx-QuickRef.htm
  • Extension - This subject acts upon the file extension of the file in question, excluding the filename and '.'.
    • is - acts upon a file if the file extension matches exactly.
    • is not - acts upon all files that do not match the file extension exactly.
    • matches one of - acts upon a file if the file extension matches any one of the elements in a comma separated list. NOTE: The list must not contain spaces between the elements and the comma, otherwise you may miss a match.
    • does not match one of - acts upon a file if the file extension does not match any one of the elements in a comma separated list. NOTE: The list must not contain spaces between the elements and the comma, otherwise you may miss a match.
    • contains - acts upon a file if the file extension contains the specified text exactly.
    • does not contain - acts upon a file if the file extension does not contained the specified text exactly.
    • contains one of - acts upon a file if the file extension contains any one of the elements in a comma separated list. NOTE: The list must not contain spaces between the elements and the comma, otherwise you may miss a match.
    • RegEx - acts upon a file if the file extension matches the Regular Expression inserted. For basic Regular Expression syntax, please used the following reference page: http://www.autohotkey.com/docs/misc/RegEx-QuickRef.htm

  • Size - This subject acts upon the file size of the file in question.
    • is - acts upon a file if the file size matches the file size specified.
    • is not - acts upon all files that do not match the file size specified .
    • is greater than - acts upon a file if the file size is greater than the file size specified.
    • is greater than or equal - acts upon a file if the file size is greater than or equal to the file size specified.
    • is less than - acts upon a file if the file size is less than the file size specified.
    • is less than or equal - acts upon a file if the file size is less than or equal to the file size specified.

Sizes are currently in units of Megabytes (MB) and Kilobytes (KB).


  • Date Last Modified - This subject acts upon the modification date of the file in question.
    • is in the last - acts upon a file if the date last modified is within the specified time period.
    • is not in the last - acts upon a file if the date last modified is not within the specified time period.
  • Date Last Opened - This subject acts upon the access date of the file in question.
    • is in the last - acts upon a file if the date last opened is within the specified time period.
    • is not in the last - acts upon a file if the date last opened is not within the specified time period.
  • Date Created - This subject acts upon the creation date of the file in question.
    • is in the last - acts upon a file if the date created is within the specified time period.
    • is not in the last - acts upon a file if the date created is not within the specified time period.

Time periods are from the current system provided time and are specified in units of minutes, hours, days, and weeks.

Rule Exclusions

The Rule Exclusion area is where the user can exclude files that match in the subject area, but with any of the selected file attributes.


The following options are available to users:

  • Read Only - Can be read, but not changed or deleted
  • Hidden - Are not shown in a directory listing by default
  • System - System files of the operating system, also not shown in a directory listing by default

Rule Verbs

The Rule Verbs area is where the actions of the rule are defined when ANY or ALL of the conditions set within the Rule Subjects area are met. The following options are available:




  • Move File - This action will relocate the file from the current location to the location set in the box to the immediate right. This location is chosen by clicking the '...' button and selecting a valid location.
    • The user may choose whether or not to Overwrite a file of a similar name in the location directory if it exists, by selecting the check box to the far right. If this box is left unchecked, and a file of the same name already exists in the destination directory, the action of this rule will be skipped, without alerting the user.
    • The user may choose whether or not to Compress a file by selecting the check box to the far right. Compression is in the form a single file zip, and takes the name of the file (minus extension) and is in the form of .zip. If the box is left unchecked, the file will be moved over as normal.
  • Move File & leave shortcut - Exactly like the option above, this action will relocate the file from the current location to the location set in the box to the immediate right. This option leaves a shortcut in the original location pointing to the new location.
    • The user may choose whether or not to Overwrite a file of a similar name in the location directory if it exists, by selecting the check box to the far right. If this box is left unchecked, and a file of the same name already exists in the destination directory, the action of this rule will be skipped, without alerting the user.
    • The user may choose whether or not to Compress a file by selecting the check box to the far right. Compression is in the form a single file zip, and takes the name of the file (minus extension) and is in the form of .zip. If the box is left unchecked, the file will be moved over as normal.

  • Rename File - This action will rename the file in the current location as identified by the conditions set above. The following naming templates (input exactly as shown below) are available for use:
    • [filename] - The full filename, minus the period and extension.
    • [fullname] - The full filename, including the period and extension.
    • [ext] - The file extension, minus the period.
    • [drive] - The file drive letter.
    • [YYYY] - Current 4-digit year (e.g. 2004).
    • [MM] - Current 2-digit month (01-12).
    • [DD] - Current 2-digit day of the month (01-31).
    • [MMMM] - Current month's full name in the current user's language, e.g. July.
    • [MMM] - Current month's abbreviation in the current user's language, e.g. Jul.
    • [DDDD] - Current day of the week's full name in the current user's language, e.g. Sunday.
    • [DDD] - Current day of the week's 3-letter abbreviation in the current user's language, e.g. Sun.
    • [WDay] - Current 1-digit day of the week (1-7). 1 is Sunday in all locales.
    • [YDay] - Current day of the year (1-366). The value is not zero-padded, e.g. 9 is retrieved, not 009.
    • [YWeek] - Current year and week number (e.g. 200453) according to ISO 8601.
    • [hh] - Current 2-digit hour (00-23) in 24-hour time (for example, 17 is 5pm).
    • [mm] - Current 2-digit minute (00-59).
    • [ss] - Current 2-digit second (00-59).
    • [ms] - Current 3-digit millisecond (000-999).
    • [DT] - The current local time in YYYYMMDDHH24MISS format.
    • [DT-UTC] - The current Coordinated Universal Time (UTC) in YYYYMMDDHH24MISS format. UTC is essentially the same as Greenwich Mean Time (GMT).

    Please note that while characters other than the templates listed above can be used, Belvedere will strip out all illegal filename characters (\, /, :, ?, <, >, *, |, ") if they are present.
    Some descriptions above were taken from the Autohotkey on line help text at http://www.autohotkey.com/docs/Variables.htm#BuiltIn . Further clarifications can be found at the aforementioned link.




  • Send file to Recycle Bin - This action will move the file from the current location to the Windows Recycle Bin.
  • Delete file - This action will irreversibly delete the file from the current location.

  • Copy File - This action will make a copy of the file in question in the location chosen by clicking the '...' button and selecting a valid location.
    • The user may choose whether or not to Overwrite a file of a similar name in the location directory if it exists, by selecting the check box to the far right. If this box is left unchecked, and a file of the same name already exists in the destination directory, the action of this rule will be skipped, without alerting the user.
    • The user may choose whether or not to Compress a file by selecting the check box to the far right. Compression is in the form a single file zip, and takes the name of the file (minus extension) and is in the form of .zip. If the box is left unchecked, the file will be moved over as normal.



  • Open File - This action will open the file in question as handled by file extension associations within your operating system. For example, if .txt files are associated with Notepad by default, the file in question will be opened by said program.
  • Print File - This action will send the file in question to the default print spooler for printing.

  • Custom - This action will execute a user defined command or executable with the matched file as first parameter of the executable.

  • Add to iTunes - This action will move the file in question to the 'Automatically add to iTunes' folder which is monitored by iTunes when it is running. This works with iTunes 9 or higher.

Rule Buttons

The final section of the Rule Creation dialog contains the dialog buttons. Here the user may Test, Save (OK) or Close without saving (Cancel) the rule creation dialog.


  • TEST - The test button will validate the input of your rule, and display the files which match the criteria set by the conditions above. No actions will be executed, nor will the rule be saved with a press of this button.
  • OK - The OK button will save your current rule settings and close the 'Create a rule...' dialog box.
  • Cancel - The Cancel button will close the 'Create a rule...' dialog box without saving the settings of the current rule, regardless of changes.
