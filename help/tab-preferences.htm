System Options

Sleeptime

The sleeptime setting is used to configure how often Belvedere will rotate through the folder and rules. After each process time, Belvedere will go to sleep for the specified period of time. Once this time period has passed, Belvedere will 'wake-up' and work through all the active rules for each folder. Time periods are from the last program execution time and are specified in units of seconds, minutes, hours, days, and weeks.

Exit Confirmation

Belvedere will display a confirmation message by default when the user tries to exit the application. This can be turned off, eliminating the need for confirmation, by removing the checkmark from this particular box.

Logging/Alert Options

System Logging

Belvedere is capable of logging all the actions within the system in 3 different forms:

  1. System Events (System) - This option allows the user to log all activity within Belvedere with respect to option changes, rule creation/deletion/activation.
  2. System Actions (Actions) - this option allows the user to log all actions that Belvedere takes based upon the rules that are set up and active within the system.
  3. All System Occurrences (Both) - This option is a combination of both of the options listed above.

Examples:

    System Events

    2009-04-28 10:38:00 Logging has been enabled with type: Both

    2009-05-01 03:28:06 SYSTEM Belvedere has been paused

    System Actions

    2009-04-28 10:38:14 ======================================
    2009-04-28 10:38:14 Action taken: Move file
    2009-04-28 10:38:14 File: C:\Documents and Settings\test\test.pdf
    2009-04-28 10:38:14 Destination: C:\Documents and Settings\test\Desktop\pdf
    2009-04-28 10:38:14 ======================================

Growl for Windows

Belvedere has the ability to send alerts to Growl for Windows (if installed) to alert users of actions as they occur. If checked, Belvedere will register itself with Growl for Windows upon startup, and send alerts. A restart is required if this setting is changed. The following types of alerts are currently supported:

  • Action - This type of alert is triggered when an acition is taken by Belvedere based on any enabled rules.
  • System - This type of alert is triggered when the user changes a system setting or configuration option.
  • Error - This type of alert is triggered when an action is taken by Belvedere by was unable to complete successfully.

Please note that Growl for Windows must be installed separately in order for this functionality to work.

Windows Tray Tips

Belvedere has the ability to send alerts to the Windows Tray to alert users of actions as they occur. If checked, Belvedere will begin sending alerts to the Windows Tray. A restart is required if this setting is changed. The following types of alerts are currently supported:

  • Action - This type of alert is triggered when an acition is taken by Belvedere based on any enabled rules.
  • System - This type of alert is triggered when the user changes a system setting or configuration option.
  • Error - This type of alert is triggered when an action is taken by Belvedere by was unable to complete successfully.

Default Rule Options

When creating a new rule, the Rule Options in the top right hand corner of the Create rule dialog can be defaulted to checked/unchecked, depending on the options chosen in this section. This setting will then apply to all newly created rules.

Command Line Parameters

Belvedere has the ability accept command line parameters to alter various settings of the application at runtime. The following parameters and their values are currently accepted by Belvedere:

    -r <integer>
    Specifies the number of times that you would like Belvedere to run and then exit quietly.
    Example: Belvedere.exe -r 5 will result in Belvedere running through all rules 5 times and then exiting quietly.

NOTE: All parameters and their corresponding values must be separated by a space in order to be accepted. If any invalid parameters or values are entered, Belvedere will run normally without any settings modified.

If System logging is enabled, both the initial acceptance of the setting and the exit event will be recorded in the event log.

Verify Configuration

By clicking this button, Belvedere will both verify and correct basic errors within the rules.ini file. This functionality is specifically used when the rules file is found to be corrupted or incomplete. This verification should also be executed immediately following an application upgrade to ensure that the rules.ini file it upgraded to the most current release.
